Lesson 1 -Basics of the Web

In this lesson, we'll cover HTML, URLs, HTTP, and Web apps.

The World Wide Web (WWW) is a collection of HTML docs (HTML = hyper text markup language), that glue everything together through the use of links (hyper links), thus creating the web. Invented in the early 90s, there are now 30+ billion pages.

Several major "pieces" of the web are inter-related and work together, including You :), a computer running a browser, the internet, and servers. The internet and servers talk to each other through http.

HTML is short for hyper text markup language, which is made up of... text content - what you can see; markup - what it looks like; references to other docs or files with associated attributes (images/videos, etc.), and links to other pages.

HTML markup is made of tags. There is an opening tag, followed by content, and then a closing tag. Each tag is named by a tag description within brackets. An entire tag is called an element. HTML attributes are tag attributes that ascribe a value, and tags can have multiple values. An example of an attribute assigned to a tag is an anchor. Anchors are for making links, like assigning a URL as an attribute to piece of content.

Image tags don't have a closing tag and are called a void tag if they don't have a closing tag, because these kinds of tag have no content. Good web coding citizenry occurs when coders assign alternate text to display when an image doesn't load, even though doing so isn't required. It's also important to remember that all line spaces must be defined, forcing breaks through the use of tags.

Inline or block tags are used to simply end a line of text, and are void tags like line break = br. Block tags make an invisible box for the text, and these can be assigned height and width. Span is an example of an inline tag that represents text, and div is an example of a block tag that behaves like a paragraph tag and creates a box around the tagged section.

There is specific nomenclature associated with defining an HTML Document. Doctype specifies the type of document at the beginning, followed by the opening doc tag, like html. This indicates the beginning of the document. After this identificaiton, attributes like head, title, etc., are included and this is where CSS guidelines would be defined. After the head and title tags, the body of the document is coded.

How Learning to Code Works

Whenever we learn anything, we begin as novices. Learning is a continuum of moving from being novices to becoming experts. The goal of this course is for learners to gain ability in most fundamental programming concepts, and awareness of many concepts beyond this class.

As learners, we go through stages of expertise, including...

Ignorance - In this stage, a novice could feel like they lack any awareness or understanding of the topic, but this is where every journey to master begins.

Awareness - Learners can move quickly from Ignorance to Awareness.

Ability - It takes a little more time to reach ability, but it is at this stage that learners begin to use their newfound, possibly still rudimentary, skills

Fluency - It takes the longest amount of time to reach ths level, but it is here that learners have reached expert status.

How the Web Works/How HTML is structured

Front-end web development is visual. You get to SEE what you develop in a browser as you create it, so there's immediate gratification and early rewards. Yay! :) HTML, CSS, and Java script have different purposes when developing a web page. Each purpose is related to how you break down a visual design to create a web page:

Structure - HTML is like the structure of a house

Design - CSS is like decorating the house... the wall color, artwork, architectural details, etc.

Modification - Java script is like any changes you make to the house, like when a garage door opener changes the structure of the house by opening the garage door

But wait, how does the browser read this? Code has a tree-like structure. Instructions for the browser are found in HTML's tree-like structure. While HTML and CSS are both languages, HTML controls the structure of the web page while CSS controls the style of the page (how it looks). When a programmer talkes about the DOM (Document Object Model), they are talking about the tree-like structure of the page - which is how the browser understands the page.

There are shared attributes as well as differences across HTML, CSS and DOMs. HTML has language, syntax and rules; as does CSS. HTML uses tags to describe content, which can be empty or contain another element, and tags can ascribe attributes and values in addition to describing content. Totally clear, right? :) Like a set of nesting dolls, the tree-like structure of HTML provides the structure and content of the page from the outermost variable to the innermost.

Similar to HTML, CSS also has language, syntax and rules, but uses it's language to designate the style aspects of the page - i.e., what style to use and where to get it. Unlike HTML and CSS, DOM doesn't get down to the level of describing the content or style of the page, but is instead the standard convention for interacting in HTML and describes the elements in the tree.

Supporting definitions:

HTML - Hypertext Markup Language

CSS - Cascading Style Sheet; style sheet language used for describing the look and formatting of a document written in a markup language

DOM - Document Object Model; a cross-platform and language independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of each document are organized in a tree structure, called a DOM tree.

Study Notes about Tools

What's the difference between scratchpad, codepen and sublime?

Scratchpad
most beginner friendly
speed of feedback: immediate feedback
ease of use: great for beginners
ease of sharing: impossible
power/professionalism: low
number of languages: 2 - HTML and CSS

Codepen
speed of feedback: 2-3 seconds
ease of use: fewer beginner friendly features
ease of sharing: easy, but requires an account
power/professionalism: very high
number of languages: 3 - HTML, CSS, JavaScript

Sublime
speed of feedback: on page refresh
ease of use: difficult in beginning
ease of sharing: difficult
power/professionalism: highest
number of languages: all * the languages

Note: For now, it's best to start by writing your code in scratchpad and write CSS inside of an element in Scratchpad. Then if you want to share your code, use Codepen. Later, we'll use Sublime to write code that we'll save and write on our computer.

How do I share code i've written in Codepen for free?

Sign up for the free codepen account, and click "create a new pen", enter your code. Then, in editor, you can pick full page option, which creates a URL that you can copy and paste and share with anyone!

How do I open code I've written in Sublime in a web browser?

There's a tiny icon at the top of the window that you can drag and drop into the browser address bar. When you drop it there, your web site will display.
